## Common Patterns

### What successful builders do:

1. **Day 1**: Clone, run, explore. Get familiar with the structure.
2. **Day 2**: Swap branding. Update colors, logo, copy.
3. **Day 3-7**: Build the core feature that makes your product unique.
4. **Week 2**: Deploy to production. Get real user feedback.
5. **Week 3+**: Iterate based on feedback. You're already live.

### What they don't do:

- Spend weeks on auth configuration
- Redesign the database schema from scratch
- Build yet another sidebar layout
- Agonize over folder structure decisions
